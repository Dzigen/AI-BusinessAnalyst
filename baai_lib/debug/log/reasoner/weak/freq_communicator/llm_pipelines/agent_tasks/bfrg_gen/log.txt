Разработка чат-бота и веб-сервиса, обеспечивающих автоматизированную поддержку заказчиков цифровых продуктов в процессе формирования корректного технического задания (ТЗ), что позволит сократить количество ошибок и недоразумений при передаче требований от заказчика к команде разработки, повысить качество подготовленных технических заданий и ускорить процесс их согласования. # Зафиксированный набор функциональных требований по данной пользовательской истории

- Система обязана предоставить пользователю удобный инструмент для непосредственного редактирования выбранного раздела или пункта ТЗ.
- Процесс редактирования должен происходить без необходимости повторного прохождения этапов обсуждения уже согласованных ранее аспектов ТЗ.

Основной задачей наставника является обеспечение эффективной внешней памяти для агентов на базе больших языковых моделей (LLM), позволяющей анализировать коммуникации, проверять предоставленную менеджерами информацию и давать рекомендации по улучшению ответов клиентам. Наставник должен поддерживать обновление и изменение графа знаний на основе структурированных и слабо структурированных данных, формировать точные ответы на запросы, а также обеспечивать валидацию и верификацию ответов менеджеров согласно установленным правилам и критериям оценки. Проект предполагает использование современных методов построения графов знаний и интеграцию различных типов данных для достижения уровня мировых стандартов (SOTA) в области разработки LLM-агентов.
# Пользовательская история (user story) в рамках заданного проекта
Специалист отдела аналитики загружает новые данные в систему цифрового наставника для обновления графа знаний, чтобы поддерживать актуальность рекомендаций и проверок.

- Специалисту предоставляется возможность подтверждения корректности предварительно обработанных данных перед началом интеграции.
- Система должна предоставлять пользователю интерфейс для загрузки новых данных, позволяющий выбирать тип данных (структурированные или слабо структурированные).
- Система должна информировать пользователя о начале процедуры предварительной обработки загруженных файлов.
- Для каждого типа данных система должна предлагать соответствующую форму загрузки: выбор файла или ручной ввод данных.
- Интерфейс системы должен позволять специалисту завершить работу с модулем загрузки данных только после успешного завершения интеграции или принятия решения о повторной загрузке.
- Система должна предоставлять интерфейс для загрузки одного или нескольких файлов специалистом аналитического отдела.
- Отчёт системы после интеграции должен содержать информацию обо всех обнаруженных ошибках и отклонениях от ожидаемого поведения.
- В случае обнаружения ошибок система должна предоставить пользователю возможность исправить некорректные данные и повторить попытку загрузки.
- При успешной проверке система автоматически обновляет существующий граф знаний новыми данными.

- Загруженные пользователем данные должны проходить автоматическую обработку и проверку на соответствие формату и структуре ожидаемых данных.

- По окончании интеграции система обязана предоставить пользователю отчёт о количестве успешно интегрированных и модифицированных элементов графа знаний.
- Предварительная обработка должна включать проверку формата и структуры данных перед их добавлением в граф знаний.

# Описание назначения проекта
Разработка программной библиотеки, представляющей собой цифрового наставника для клиентских менеджеров банка, способного анализировать коммуникации менеджеров с клиентами, проверять предоставленную менеджерами информацию на соответствие базе знаний банка и давать рекомендации по улучшению взаимодействия с клиентами. Система включает механизмы обновления и изменения графа знаний, обеспечивающего хранение технической и сопроводительной информации о продуктах банка, правил общения с клиентами и истории текущих диалогов. Основной целью разработки является повышение квалификации менеджеров, снижение оттока клиентов и увеличение лояльности путем улучшения качества обслуживания и повышения эффективности коммуникаций. Библиотека обеспечит функции валидации ответов менеджеров, корректировки этих ответов и формирования рекомендаций возможных ответов в процессе текущего диалога с клиентом.
# Пользовательская история (user story) в рамках заданного проекта
Система формирует пространство памяти на основе графов знаний, храня текущее состояние диалога, правила общения с клиентами и техническую информацию о банковских продуктах, обеспечивая контекстно-зависимые ответы менеджеру.
# Зафиксированный набор функциональных требований по данной пользовательской истории
- После принятия изменений системой должен обновляться текущий контекст диалога.
- Система должна предоставлять возможность запроса источников данных для построения графа знаний от акторов.
- Система должна формировать пространство памяти на основе графов знаний, включающее текущее состояние диалога, правила общения с клиентами и техническую информацию о банковских продуктах.
- Система должна извлекать текущие данные диалога менеджера с клиентом.
- При наличии ошибок или несоответствий во введённом тексте система должна предлагать скорректированный вариант ответа.
- При получении дополнительных данных или инструкций по изменениям структура графа знаний должна обновляться системой.
- Акторы должны иметь возможность подтверждения правильности сформированного графа знаний или отправки запросов на внесение изменений.
- Система должна создавать новый узел в графе знаний при начале нового диалога.
- После завершения обработки данных система должна уведомлять акторов о готовности графа знаний.
- Система должна принимать от акторов базы знаний банка, технические документы продуктов и правила общения с клиентами в качестве источников данных.
- Система должна регистрировать ответы клиента, добавляя их в текущий узел диалога и обновляя граф знаний соответствующим образом.
- Система должна позволять менеджеру закрывать диалог по завершении общения с клиентом.
- Процедура формирования графа знаний считается завершённой только после получения положительного подтверждения от актора.
- Система должна сохранять завершенные диалоги в графе знаний и освобождать ресурсы, связанные с этим диалогом.
- Система должна анализировать введённый менеджером текст сообщения клиента и сопоставлять его с правилами общения и базой знаний банка.
- При соответствии введённого текста сообщению правилам общения и базе знаний банка система должна подтверждать корректность ответа.
- Система должна обрабатывать полученные источники данных и формировать граф знаний.
- Система должна сохранять текущее состояние диалога в соответствующем узле графа знаний.
- Системой должна поддерживаться возможность повторного прохождения процесса анализа и коррекции сообщений при продолжении диалога с клиентом.
- Системе следует обеспечивать возврат обновлённой версии графа знаний акторам после внесения изменений.
- Система должна обеспечивать возможность контекстно-зависимых подсказок и рекомендаций менеджеру на основании сохраненного состояния диалога.
- По запросу акторов система должна возвращать актуальную версию графа знаний.
- Система должна предоставлять возможность менеджеру принять предложенные исправления или отклонить их.
- Система должна добавлять текст сообщений менеджера в текущий узел диалога и обновлять граф знаний соответственно.
- Система должна принимать от менеджера текст сообщения клиенту либо выбор варианта из предложенного набора сообщений.
- Система должна динамически обновлять граф знаний после каждого обмена сообщениями, поддерживая актуальность состояния диалога.
