- Личный кабинет должен содержать раздел "Чат-бот", позволяющий инициировать диалог с системой.
- Окно чата должно автоматически открываться после выбора раздела "Чат-бот" из меню личного кабинета.
- Интерфейс окна чата должен позволять заказчику отправлять текстовые сообщения с запросами и получать ответные сообщения от системы.

# Описание назначения проекта
Разработка чат-бота и веб-сервиса, обеспечивающих автоматизированную поддержку заказчиков цифровых продуктов в процессе формирования корректного технического задания (ТЗ), что позволит сократить количество ошибок и недоразумений при передаче требований от заказчика к команде разработки, повысить качество подготовленных технических заданий и ускорить процесс их согласования. Система должна функционировать на базе GigaChat и low-code платформы партнера, поддерживать ведение диалога с пользователем для уточнения требований, возможность интерактивного редактирования ТЗ как через интерфейс чата, так и через облачный документ, а также обеспечивать экспорт сформированного документа в форматах PDF и DOCX. Целевой аудиторией являются заказчики цифровых решений, нуждающиеся в качественной подготовке технической документации перед началом реализации проектов.
# Заголовок (название) заданной группы функциональных требований к данному проекту
Формирование и просмотр итогового ТЗ
# Группа (набор) исходных функциональных требований, которую необходимо переформулировать
- Чат-бот должен сохранять текущее состояние технического задания и предоставлять пользователю ссылку на доступный онлайн-документ или файл для скачивания.
- Чат-бот должен предоставлять пользователю возможность просмотра предварительного варианта технического задания.
- Чат-бот должен генерировать предварительную версию технического задания на основе предоставленной пользователем информации.
- Пользователю должна предоставляться возможность запросить финальный вариант ТЗ после окончания всех необходимых правок.
- Финальная версия ТЗ должна формироваться системой по запросу пользователя и предлагаться для скачивания в одном из поддерживаемых форматов — PDF или DOCX.
- Файл должен соответствовать стандарту выбранного формата (PDF или DOCX), включая правильное отображение структуры данных и сохранение всех элементов оформления технического задания.
- Форматированный текст и изображения, содержащиеся в техническом задании, сохраняются в исходном виде при экспорте в оба указанных формата.
- Сгенерированный файл соответствует выбранному формату и содержит актуальную версию технического задания со всеми внесенными изменениями и дополнениями.
- После сохранения система информирует пользователя об успешной операции и предоставляет ему доступ к просмотру финального варианта технического задания с учётом всех сделанных изменений и дополнений.
- Сохраненное техническое задание должно быть доступно для последующего экспорта в форматы PDF и DOCX.

# Описание назначения проекта
Проект направлен на разработку минимально жизнеспособного продукта (MVP) — цифрового наставника, предназначенного для повышения качества взаимодействия менеджеров технической поддержки банка с клиентами. Основной задачей наставника является обеспечение эффективной внешней памяти для агентов на базе больших языковых моделей (LLM), позволяющей анализировать коммуникации, проверять предоставленную менеджерами информацию и давать рекомендации по улучшению ответов клиентам. Проект предполагает использование современных методов построения графов знаний и интеграцию различных типов данных для достижения уровня мировых стандартов (SOTA) в области разработки LLM-агентов.

- Подтвержденный и проверенный ответ должен быть доступен для отправки клиенту непосредственно через систему.

- Система должна позволять менеджеру просмотреть предложенные варианты рекомендаций перед выбором одного из них.

- Если менеджер вводит собственный ответ, система должна предложить ему возможные дополнения или улучшения на основе введённого текста.

# Описание назначения проекта
Проект направлен на разработку минимально жизнеспособного продукта (MVP) — цифрового наставника, предназначенного для повышения качества взаимодействия менеджеров технической поддержки банка с клиентами. Основной задачей наставника является обеспечение эффективной внешней памяти для агентов на базе больших языковых моделей (LLM), позволяющей анализировать коммуникации, проверять предоставленную менеджерами информацию и давать рекомендации по улучшению ответов клиентам. Наставник должен поддерживать обновление и изменение графа знаний на основе структурированных и слабо структурированных данных, формировать точные ответы на запросы, а также обеспечивать валидацию и верификацию ответов менеджеров согласно установленным правилам и критериям оценки. Проект предполагает использование современных методов построения графов знаний и интеграцию различных типов данных для достижения уровня мировых стандартов (SOTA) в области разработки LLM-агентов.
# Заголовок (название) заданной группы функциональных требований к данному проекту
Контроль и проверка информации
# Группа (набор) исходных функциональных требований, которую необходимо переформулировать
- После внесения изменений система должна предоставить возможность повторной инициации проверки ответа через интерфейс цифрового наставника.
- Все этапы процесса проверки и коррекции ответа должны логироваться системой для последующего анализа и аудита.
- В случае обнаружения несоответствия или ошибок в ответе, цифровой наставник должен автоматически генерировать рекомендации по их устранению.
- Цифровой наставник обязан проводить семантический анализ текста ответа менеджера.
- При обнаружении несоответствий или потенциальных ошибок цифровой наставник должен автоматически генерировать рекомендации по исправлению ответа.
- Цифровой наставник обязан сравнивать полученный от менеджера ответ с информацией из графа знаний и установленными правилами корректности.
- Система обязана сравнивать проанализированный ответ с данными из базы знаний и установленными правилами предоставления информации.
- Цифровой наставник обязан подтвердить правильность финальной версии ответа после повторного прохождения процедуры проверки.
- В случае успешного прохождения проверки система должна подтвердить готовность ответа для отправки клиенту.

# Описание назначения проекта
Разработка программной библиотеки, представляющей собой цифрового наставника для клиентских менеджеров банка, способного анализировать коммуникации менеджеров с клиентами, проверять предоставленную менеджерами информацию на соответствие базе знаний банка и давать рекомендации по улучшению взаимодействия с клиентами. Система включает механизмы обновления и изменения графа знаний, обеспечивающего хранение технической и сопроводительной информации о продуктах банка, правил общения с клиентами и истории текущих диалогов. Основной целью разработки является повышение квалификации менеджеров, снижение оттока клиентов и увеличение лояльности путем улучшения качества обслуживания и повышения эффективности коммуникаций. Библиотека обеспечит функции валидации ответов менеджеров, корректировки этих ответов и формирования рекомендаций возможных ответов в процессе текущего диалога с клиентом.

# Описание назначения проекта
Разработка программной библиотеки, представляющей собой цифрового наставника для клиентских менеджеров банка, способного анализировать коммуникации менеджеров с клиентами, проверять предоставленную менеджерами информацию на соответствие базе знаний банка и давать рекомендации по улучшению взаимодействия с клиентами. Система включает механизмы обновления и изменения графа знаний, обеспечивающего хранение технической и сопроводительной информации о продуктах банка, правил общения с клиентами и истории текущих диалогов. Основной целью разработки является повышение квалификации менеджеров, снижение оттока клиентов и увеличение лояльности путем улучшения качества обслуживания и повышения эффективности коммуникаций. Библиотека обеспечит функции валидации ответов менеджеров, корректировки этих ответов и формирования рекомендаций возможных ответов в процессе текущего диалога с клиентом.
# Заголовок (название) заданной группы функциональных требований к данному проекту
Проверка и автоматический контроль соответствия сообщениям
# Группа (набор) исходных функциональных требований, которую необходимо переформулировать
- Система должна обеспечивать возможность ожидания менеджером завершения процесса проверки введенного им ответа.
- Процесс проверки должен повторяться каждый раз после внесения изменений до достижения соответствия требованиям.
- Система должна информировать менеджера о результате сравнения введённого ответа с базой знаний и правилами общения.
- Процесс проверки ответа должен продолжаться до тех пор, пока менеджер не получит положительный результат проверки.
- В случае выявления ошибок система должна предоставить конкретные уведомления о характере ошибок и рекомендации по их устранению.
- Система должна предоставлять интерфейс для инициации процесса проверки введённого текста.
- Система должна сравнивать введённый ответ с актуальной версией базы знаний банка и актуальными правилами общения с клиентами.
- В случае обнаружения ошибок или несоответствий во введённом ответе, система должна уведомлять менеджера о необходимости редактирования ответа.
- Система должна сравнивать введённый текст с данными из базы знаний банка.
- Система должна информировать клиента-менеджера о результате сравнения: успехе или наличии ошибок.
- Система должна проводить автоматическую валидацию самостоятельного ответа менеджера на предмет соответствия базе знаний банка.
- Система должна подтверждать отправку ответа клиенту только после успешного прохождения процедуры валидации.
- Система должна автоматически создавать новую сессию общения при начале нового взаимодействия с клиентом.
- Система должна обеспечивать завершение сессии общения после завершения взаимодействия с клиентом.
- Система должна проводить автоматический предварительный анализ введённого менеджером текста на предмет его соответствия актуальным данным из базы знаний банка и установленным правилам общения с клиентами.
- Система должна проводить автоматическую предварительную проверку введённого менеджером текста сообщения перед отправкой клиенту на предмет наличия ошибок и несоответствий требованиям базы знаний банка.
- Система должна анализировать введённый менеджером текст сообщения клиента и сопоставлять его с правилами общения и базой знаний банка.
- При соответствии введённого текста сообщению правилам общения и базе знаний банка система должна подтверждать корректность ответа.

- В случае отрицательного результата проверки система должна блокировать отправку сообщения и требовать от пользователя внести необходимые правки.
- По завершении успешной проверки система должна разрешить отправку сообщения клиенту и завершить текущий сеанс проверки.
- При каждом новом взаимодействии с клиентом система должна автоматически открывать новую сессию общения и сохранять историю предыдущих взаимодействий.
- Завершение текущей сессии общения должно происходить автоматически системой после подтверждения факта завершения взаимодействия с клиентом.
- Автоматическая проверка введённого текста должна проводиться исключительно на основе актуальной версии базы знаний банка и действующих правил общения с клиентами.
- Проверка должна выполняться таким образом, чтобы её результаты были однозначно интерпретируемыми и легко проверяемыми.
